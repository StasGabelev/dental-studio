# 🦷 Dental Studio - Omnichannel AI Hub (v2.0)

Этот репозиторий содержит систему мультиплатформенной ИИ-поддержки для клиники Dental Studio.

## 📁 Структура новых файлов:
1. `omni-bot-server.js` — Центральный сервер (Node.js) для Telegram, Viber и Веб-виджета.
2. `package.json` — Список зависимостей для сервера.
3. `messenger_update.sql` — SQL-скрипт для обновления базы данных Supabase.
4. `chat-widget.js` — Обновленный виджет сайта с выбором платформы.
5. `admin/` — Обновленная админка с полями для всех мессенджеров.

---

## 🚀 Инструкция по запуску:

### 1. База данных (Supabase)
Зайдите в **SQL Editor** своего Supabase и выполните код из файла `messenger_update.sql`. 
Это добавит в таблицу `ai_settings` нужные поля:
- `tg_patient_bot_token` (для пациентов)
- `viber_bot_token`
- `wa_bot_token`
- `wa_link`

### 2. Установка сервера (VPS)
Перенесите `omni-bot-server.js` и `package.json` на свой сервер Hostinger и выполните команды:

```bash
npm install
pm2 start omni-bot-server.js --name "dental-studio-ai"
```

**Важно**: Обязательно добавьте переменную окружения `SUPABASE_KEY` (ваш service_role ключ) на сервере.

### 3. Настройка в Админке
Зайдите в раздел **«Налаштування ІІ»** на сайте:
1. В левом блоке введите токен бота для **Админ-уведомлений**.
2. В правом блоке введите токен для **Клиентского бота Telegram** и **Viber**.
3. Сохраните.

---

## 💡 Как это работает:
- Пациент на сайте видит выбор: Telegram, Viber, WhatsApp или Чат на сайте.
- При выборе мессенджера он попадает к соответствующему боту.
- На все вопросы отвечает один и тот же ИИ, используя вашу базу знаний из Supabase.
- Если ИИ не справляется («тупиковая ситуация»), он шет уведомление админу.
