# 🏥 Dental Studio - Project Passport (Admin & AI)

## 📌 Общий обзор
Этот проект — современная панель управления клиникой "Dental Studio", построенная на базе **SPA (Single Page Application)** с использованием чистого JavaScript и **Supabase** в качестве бэкенда.

## 🚀 Текущий статус
- **Последняя стабильная версия**: **v1.31.40**
- **Главный файл логики**: `admin/admin_v3.js`
- **Главный файл интерфейса**: `admin/panel.html`

## 🛠 Архитектура и правила (ДЛЯ ИИ)
1. **Навигация**: Используется механизм `switchSection(sectionId)`. Все переходы работают через хэш в URL (`#dashboard`, `#portfolio` и т.д.). Категорически нельзя удалять ID секций (`sec-dashboard`, `sec-portfolio`).
2. **Версионирование**: Чтобы избежать проблем с кэшом, после каждой правки нужно обновлять версию в `panel.html` в двух местах: в бейдже версии и в ссылке на скрипт (`admin_v3.js?v=v1.31.XX`).
3. **Редактор Кейсов**: Фото "До" и "После" вынесены в отдельный блок **"РЕЗУЛЬТАТ"** под этапами лечения.
4. **Supabase**: Соединение инициализируется глобально через объект `sb`. Ключи хранятся в `localStorage` (браузер) и в таблице после авторизации.

## 🔑 Хранение настроек и ключей
Все чувствительные данные хранятся в таблице **`ai_settings`** в Supabase:
- **Telegram**: `tg_bot_token`, `tg_chat_id`, `tg_patient_bot_token`.
- **Viber/WhatsApp**: `viber_bot_token`, `wa_bot_token`, `wa_link`.
- **AI (LLM)**: `api_key`, `provider`, `model`, `system_prompt`, `knowledge_base_manual`.

## 📦 База данных (Supabase)
Основные таблицы:
- `treatment_cases`: Хранит портфолио. Поля: `slug`, `title_uk`, `before_text`, `after_text`, `stages` (JSON).
- `ai_settings`: Конфигурация ботов и ИИ.
- `site_pages`: Контент страниц сайта.
- `price_items`: Прайс-лист.
- `doctors`: Карточки врачей.

## ⚠️ Важное предупреждение
Если навигация в админке ломается (ничего не нажимается) — не пытайся патчить `admin_v3.js` частично. **Откатывайся на стабильную базу** (последний стабильный коммит) и только потом делай точечные изменения.
